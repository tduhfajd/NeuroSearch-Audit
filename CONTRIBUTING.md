# Contributing to NeuroSearch Analyzer

Спасибо за интерес к проекту.

## Как предложить изменения

1. Сделайте fork репозитория.
2. Создайте отдельную ветку:
   - `feature/<short-name>` для новой функциональности
   - `fix/<short-name>` для исправлений
3. Внесите изменения небольшим и понятным diff.
4. Проверьте, что проект запускается локально.
5. Откройте Pull Request в `main`.

## Локальная проверка перед PR

Минимально рекомендуется:

```bash
make check
python run_audit.py --help
```

Если изменяли установку/документацию, проверьте соответствующие инструкции из `README.md`.

## Стиль изменений

- Делайте маленькие атомарные коммиты с понятным смыслом.
- Не смешивайте рефакторинг и новую функциональность в одном PR без необходимости.
- Обновляйте документацию, если меняете пользовательский сценарий.
- Добавляйте примеры использования, когда это помогает быстрее понять изменение.

## Сообщения об ошибках и идеи

- Для багов и предложений используйте Issues.
- В баг-репорте укажите:
  - шаги воспроизведения
  - ожидаемое поведение
  - фактическое поведение
  - окружение (OS, версии Python/Node/Go)
