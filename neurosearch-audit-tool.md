# NeuroSearch Audit — Внутренний инструмент

> **Статус:** Концепция / Предпроектная документация  
> **Платформа:** macOS (Apple Silicon M1/M2/M3/M4)  
> **AI-движок:** ChatGPT Plus (web, без API)  
> **Аудитория:** Внутреннее использование — пресейл, аудит, генерация КП

---

## 1. Суть инструмента

**NeuroSearch Audit** — локальный инструмент на Mac, который автоматизирует SEO-аудит сайтов с оценкой их готовности к AI-ответам (Яндекс Нейро, ChatGPT, GigaChat).

Инструмент делает три вещи:
1. **Сканирует** сайт и конкурентов (краулер)
2. **Анализирует** техническую и AI-готовность (правила + LLM)
3. **Генерирует** отчёт + план работ + КП (PDF/HTML)

Ключевая идея: AI-анализ работает через **ChatGPT Plus через браузер** — без API-ключей, без лишних трат. Подключаешь свой аккаунт, инструмент сам отправляет запросы и получает ответы.

## 2. Ограничения и решения

| Ограничение | Решение |
|---|---|
| Нет API-ключа ChatGPT | Браузерная автоматизация через Playwright (см. раздел 5) |
| Mac Apple Silicon | Python arm64 + нативные пакеты, всё работает нативно |
| Внутренний инструмент (не SaaS) | Локальный веб-сервер, UI в браузере, никакого деплоя |
| Нельзя гарантировать "место в ChatGPT" | Честные метрики: "вероятность цитирования", не "гарантия" |

## 3. Что анализирует инструмент

### 3.1 Технический SEO (под Яндекс + Google)

**Индексация и структура:**
- `robots.txt`, `sitemap.xml`, `canonical`, `noindex`
- Статус-коды (200/301/302/404/500), цепочки редиректов
- Дубли title/h1/description
- Структура URL, хлебные крошки

**Скорость и доступность:**
- Core Web Vitals (LCP, INP, CLS) через PageSpeed API
- Мобильная версия
- Турбо-страницы Яндекса (проверка наличия)

**Структурированные данные:**
- JSON-LD: Organization, Product, FAQ, HowTo, Article, BreadcrumbList
- Корректность и полнота полей

**Коммерческие сигналы доверия (Яндекс E-E-A-T):**
- Контакты, реквизиты, политика конфиденциальности
- Страница "О компании", авторы материалов
- Гарантии, отзывы

### 3.2 AI-готовность (GEO 2.0 / AEO)

Оценка того, насколько контент пригоден для извлечения нейросетями:

| Метрика | Что проверяем |
|---|---|
| **Answer Format Score** | Есть ли краткий прямой ответ в первых абзацах |
| **Structure Density Score** | h2/h3, списки, таблицы, шаги, нумерация |
| **Definition Coverage** | Определены ли ключевые термины сайта |
| **Authority Signals Score** | Авторы, кейсы, цифры, ссылки на источники |
| **Schema Completeness Score** | FAQPage, HowTo, Article корректны и полны |

**Итоговый AI Visibility Readiness Index:**
```
AVRI = (Answer×0.25) + (Structure×0.20) + (Definition×0.20) + (Authority×0.20) + (Schema×0.15)
```
_Диапазон: 0–100. Это не гарантия попадания в AI-ответы — это оценка вероятности корректного цитирования._

## 4. Вход и выход

### Вход
```
- Домен сайта
- Ниша / регион / язык
- Цель: лиды / инфо-трафик / локальные услуги / B2B
- Конкуренты (3–10, можно авто-подбором)
```

### Выход (то, за что платят клиенты)
```
1. Дашборд: SEO Score + AI Visibility Score + Tech Health
2. Карта проблем: P0 критично / P1 важно / P2 улучшение / P3 задел
3. Рекомендации: конкретные правки по страницам
4. Контент-план: что переписать / создать
5. PDF-отчёт: для клиента
6. КП: 2–3 пакета с ценами и описанием работ
```

## 5. ChatGPT Plus без API — как это работает

Вместо платного API инструмент использует **браузерную автоматизацию Playwright**:

```
Инструмент → Playwright → chat.openai.com → твой аккаунт Plus → ответ ChatGPT → парсинг → анализ
```

### Принцип работы

1. При первом запуске — открывается браузер, ты логинишься в ChatGPT вручную (один раз)
2. Playwright сохраняет сессию (`storage_state.json`) на диск
3. При каждом аудите — инструмент автоматически открывает headless-браузер, отправляет промпты, парсит ответы
4. Ты не видишь браузер — всё работает в фоне

### Ограничения браузерного подхода

| Ограничение | Решение |
|---|---|
| ChatGPT иногда меняет UI | Playwright селекторы можно обновить (один файл) |
| Rate limit Plus: ~50 сообщ/3ч | Очередь запросов с задержками, батчинг страниц |
| Сессия истекает | Автопроверка сессии перед каждым аудитом |
| Нет параллельности | Последовательная обработка, но быстро (секунды на страницу) |

### Альтернатива (если нужно больше запросов)
Подключить **Ollama** с локальной моделью (llama3, mistral) — работает нативно на Apple Silicon, полностью бесплатно, без ограничений. Легко переключить в настройках.

## 7. Стек технологий

```
Runtime:       Python 3.12 (arm64, нативный для M-series)
Backend:       FastAPI + Uvicorn
Crawler:       Scrapy 2.x + playwright (chromium arm64)
AI-Bridge:     Playwright (Python) → chat.openai.com
Local AI opt.: Ollama (llama3.2, mistral) — опционально
Database:      PostgreSQL 16 (через Homebrew или Docker)
ORM:           SQLAlchemy 2 + Alembic
Task Queue:    Redis + RQ (лёгкая альтернатива Celery)
Reports:       Jinja2 templates + WeasyPrint (PDF)
Frontend:      HTML + Tailwind CSS + Alpine.js (минимально)
Package mgr:   uv (быстрый, нативный для arm64)
```

**Почему это хорошо работает на M-series:**
- Python arm64 — нативная скорость, нет Rosetta
- Chromium через Playwright — есть нативная arm64 сборка
- Ollama — специально оптимизирован под Apple Neural Engine
- PostgreSQL — нативный через Homebrew
- WeasyPrint — работает без issues через pip arm64

## 9. Приоритеты проблем (как классифицируем)

```
P0 — КРИТИЧНО (блокирует)
  • сайт закрыт от индексации (robots.txt, noindex на всём)
  • каскадные редиректы, петли
  • критические ошибки скорости (<20 PageSpeed)

P1 — ВАЖНО (серьёзно мешает)
  • дубли страниц без canonical
  • отсутствие мобильной версии
  • нет sitemap / битые ссылки в sitemap
  • LCP > 4s на ключевых страницах

P2 — AI-ГОТОВНОСТЬ (новая услуга)
  • нет FAQ/HowTo разметки
  • нет прямых ответов на интенты
  • нет определений терминов
  • нет кейсов/авторов/источников

P3 — УЛУЧШЕНИЯ (задел на рост)
  • дополнительные схемы (Review, VideoObject)
  • углубление внутренней перелинковки
  • расширение тематических кластеров
```

## 10. Генерация КП — логика

Инструмент автоматически формирует коммерческое предложение по результатам аудита:

```
[Данные аудита] → [Группировка проблем] → [Выбор пакета] → [Генерация текста (ChatGPT)] → [PDF]
```

### Пакеты

**Start** — техническое здоровье
> Исправление P0+P1 проблем, базовая schema-разметка, устранение дублей.
> _Для: сайты с техническими проблемами, которые мешают индексации._

**Growth** — контент и AI-готовность
> Start + реструктуризация контента под интенты, FAQ/HowTo разметка, 10–20 страниц переработки.
> _Для: сайты, которые хотят попадать в нейро-ответы Яндекса._

**Authority** — экспертность и brand signals
> Growth + кейсы, авторские материалы, whitepaper, мониторинг упоминаний.
> _Для: B2B, медицина, юриспруденция — там где E-E-A-T критичен._

### Честные формулировки (важно)

```
❌ Нельзя: "Вы будете в ответах ChatGPT"
✓ Можно:  "Контент станет пригодным для извлечения и цитирования нейросистемами"

❌ Нельзя: "Гарантируем ТОП-1 в Яндексе"
✓ Можно:  "Устраняем технические барьеры, которые мешают индексации и ранжированию"
```

## 12. Быстрый старт (план установки)

```bash
# 1. Клонировать / создать проект
mkdir neurosearch-audit && cd neurosearch-audit

# 2. Создать виртуальное окружение (uv, нативный arm64)
uv venv && source .venv/bin/activate

# 3. Установить зависимости
uv add fastapi uvicorn scrapy playwright sqlalchemy alembic \
       redis rq weasyprint jinja2 psycopg2-binary python-dotenv

# 4. Установить Playwright браузеры (arm64)
playwright install chromium

# 5. Запустить PostgreSQL (через Homebrew)
brew services start postgresql@16

# 6. Первый запуск — авторизация в ChatGPT (один раз)
python backend/analyzer/ai_bridge.py --setup

# 7. Запустить backend
uvicorn backend.main:app --reload --port 8000

# 8. Открыть в браузере
open http://localhost:8000
```
