## Правила работы агента

### Разрешено

- Читать файлы, искать по репо, объяснять код.
- Выполнять задачи строго по текущему PLAN.md.
- Делать **минимальный diff** в рамках одной задачи.
- Добавлять/обновлять тесты рядом с изменением.
- Запускать проверки и чинить найденное.
- Коммитить каждую задачу отдельно.

### Запрещено без явного подтверждения

- Выход за scope текущего плана/фазы.
- Массовые рефакторинги, переименования.
- Изменения БД/auth/security/infra вне спецификации.
- Удаления файлов, `git reset`, разрушительные действия.
- Подключение новых зависимостей.

### Лимиты

- Одна задача за итерацию.
- Max 2 запуска тяжёлых тестов.
- Тесты падают 2 раза подряд по разным причинам → **остановиться**.

### Stop-policy

Остановись если:
- Не хватает требований.
- Риск для данных/безопасности.
- Задача выходит за scope.
- Нужна установка пакетов.
- Баг вне scope → зафиксировать, не чинить.

## Workflow выполнения задачи

### Перед началом
1. Прочитать `AGENTS.md` (этот файл).
2. Прочитать `.planning/STATE.md`.
3. Прочитать текущий PLAN.md.

### В ходе выполнения
1. Минимальный diff.
2. Запустить `<verify>` из задачи.
3. Починить если нужно.
4. Коммит: `<type>(<phase>-<plan>): <описание>`.

### После выполнения
1. Создать SUMMARY.md.
2. Обновить STATE.md.
3. Обновить CHANGELOG.md.
4. Баги вне scope → `.planning/todos/pending/`.
