# Требования: REQUIREMENTS.md

REQUIREMENTS.md — проверяемые требования, которые определяют "готово".
Каждое требование атомарно, тестируемо и привязано к фазе роадмапа.

---

## Формат требования

```
- [ ] **AUTH-01**: Пользователь может зарегистрироваться по email и паролю
```

- **ID:** `[КАТЕГОРИЯ]-[НОМЕР]` (AUTH-01, CONT-02, SOCL-03).
- **Описание:** ориентировано на пользователя, тестируемо, атомарно.
- **Чекбокс:** только для v1 требований (v2 — ещё не actionable).

## Категории

Выводятся из домена проекта:
- **AUTH** — аутентификация.
- **PROF** — профили.
- **CONT** — контент.
- **SOCL** — социальные функции.
- **NOTF** — уведомления.
- Или любые другие, уместные для проекта.

## v1 vs v2

- **v1:** обязательный scope, будет в фазах роадмапа.
- **v2:** отложено на будущее, не в текущем роадмапе.
- Перенос v2 → v1 требует обновления роадмапа.

## Out of Scope

Явные исключения с обоснованием. Предотвращает "а почему не включили X?":

| Feature | Причина |
|---------|---------|
| Real-time chat | Высокая сложность, не core value |
| OAuth login | Email/пароль достаточно для v1 |

## Трассировка (Traceability)

Каждое требование привязывается к фазе роадмапа:

| Requirement | Phase | Status |
|-------------|-------|--------|
| AUTH-01 | Phase 1 | Pending |
| AUTH-02 | Phase 1 | Pending |
| CONT-01 | Phase 3 | Pending |

Незамапленные требования = дыра в роадмапе.

## Жизненный цикл

### После каждой фазы:
1. Покрытые требования → Complete.
2. Обновить статус трассировки.
3. Зафиксировать изменения scope.

### Завершение требования:
- Фича реализована.
- Фича верифицирована (тесты пройдены, ручная проверка).
- Фича закоммичена.
