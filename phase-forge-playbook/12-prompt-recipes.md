# Готовые промпты: копируй и используй

Промпты для каждого этапа Phase Forge. Подставляй свои данные
и ссылки на файлы проекта.

---

## Промпт 1: Инициализация проекта

```
Ты — инженерный агент. Инициализируй проект по методологии Phase Forge.

Вот описание проекта:
neurosearch-audit-tool.md

Создай следующие артефакты в директории .planning/:

1. PROJECT.md по шаблону phase-forge-playbook/templates/PROJECT.md:
   - Задай мне до 10 уточняющих вопросов (только критические).
   - Зафиксируй видение, Core Value, ограничения.

2. REQUIREMENTS.md по шаблону phase-forge-playbook/templates/REQUIREMENTS.md:
   - Раздели на v1 (обязательно), v2 (потом), Out of Scope.
   - Каждое требование: атомарное, тестируемое, user-centric.

3. ROADMAP.md по шаблону phase-forge-playbook/templates/ROADMAP.md:
   - 5-8 фаз для стандартного проекта.
   - Каждая фаза: Goal, Dependencies, Requirements, Success Criteria.
   - Трассировка: каждое v1 requirement замаплено на фазу.

4. STATE.md по шаблону phase-forge-playbook/templates/STATE.md:
   - Инициализируй позицию: Phase 1 ready to plan.

Ничего не реализуй в коде. Только артефакты планирования.
Покажи результат и дождись одобрения роадмапа.
```

## Промпт 2: Создание AGENTS.md

```
Прочитай .planning/PROJECT.md и .planning/ROADMAP.md.
Создай AGENTS.md в корне репозитория по шаблону
phase-forge-playbook/templates/AGENTS.md.

Включи:
- Описание проекта и стека.
- Структуру репозитория (или планируемую).
- Маршруты/API контракт (если применимо).
- Модель данных (если применимо).
- Правила работы агента (из phase-forge-playbook/02-agent-policy.md).
- Workflow выполнения задачи.
- Quality gates.
- Ключевые соглашения (стиль, тесты, конвенции).
- Что НЕ входит в scope.
```

## Промпт 3: Обсуждение фазы (Discuss)

```
Прочитай AGENTS.md и .planning/STATE.md.
Открой .planning/ROADMAP.md — найди фазу 1: Phase 1: Foundation.

Проведи обсуждение реализации этой фазы:
1. Проанализируй фазу и выяви "серые зоны" — области,
   где возможны разные подходы.
2. Для каждой зоны задай мне конкретные вопросы с вариантами.
3. Зафиксируй мои ответы.

Результат сохрани в:
.planning/phases/[XX-name]/[XX]-CONTEXT.md
по шаблону phase-forge-playbook/templates/CONTEXT.md.

Включи: Implementation Decisions, Claude's Discretion,
Specific Ideas, Deferred Ideas.
```

## Промпт 4: Планирование фазы (Plan)

```
Прочитай AGENTS.md и .planning/STATE.md.
Прочитай .planning/ROADMAP.md — фаза 1.
Прочитай .planning/phases/01-foundation/01-CONTEXT.md.
Прочитай .planning/REQUIREMENTS.md.

Создай планы выполнения для фазы 1:

1. При необходимости проведи исследование домена
   и сохрани в [XX]-RESEARCH.md.

2. Создай 2-3 плана (PLAN.md) по шаблону
   phase-forge-playbook/templates/PLAN.md:
   - Каждый план: frontmatter + задачи в XML.
   - Волны (wave) для параллельного выполнения.
   - Зависимости (depends_on) между планами.
   - must_haves из Success Criteria фазы.
   - Верификация (<verify>) для каждой задачи.

3. Проверь планы:
   - Все v1 requirements фазы покрыты?
   - Задачи конкретные и actionable?
   - Размер: 2-3 задачи на план, не больше.

Покажи результат. Ничего не реализуй до подтверждения.
```

## Промпт 5: Выполнение фазы (Execute)

```
Прочитай AGENTS.md и .planning/STATE.md.
Прочитай все PLAN.md файлы фазы 1 из .planning/phases/01-foundation/.

Выполняй планы по волнам:
1. Волна 1: планы с wave: 1 (параллельно/последовательно).
2. Волна 2: планы с wave: 2 (после завершения волны 1) и так далее.

Для каждой задачи:
1) Реализуй минимальный diff.
2) Запусти <verify> из задачи.
3) Почини если нужно (max 2 попытки).
4) Коммит: <type>(<phase>-<plan>): <описание>.
5) Следующая задача.

После каждого плана:
- Создай SUMMARY.md по шаблону phase-forge-playbook/templates/SUMMARY.md.
- Обнови STATE.md.

После всех планов:
- Запусти полный набор тестов.
- Обнови ROADMAP.md (прогресс фазы).
- Обнови CHANGELOG.md.

Если тесты падают 2 раза подряд по разным причинам — остановись и спроси.
Баги вне scope → .planning/todos/pending/.
```

## Промпт 6: Верификация фазы (Verify)

```
Прочитай AGENTS.md и .planning/STATE.md.
Прочитай все SUMMARY.md файлы фазы 1.
Прочитай Success Criteria фазы 1 из ROADMAP.md.

Проведи верификацию:

1. Goal-backward проверка:
   - Для каждого Success Criteria → покажи evidence из кода.
   - Проверь: Truths (поведения), Artifacts (файлы), Wiring (связи).

2. Проверка стабов:
   - Все файлы содержат реальный код (не TODO, не pass, не NotImplementedError).

3. Подготовь UAT список:
   - Извлеки тестируемые поведения.
   - Представь мне по одному для ручной проверки.

Результат сохрани в:
- .planning/phases/[XX-name]/[XX]-VERIFICATION.md
- .planning/phases/[XX-name]/[XX]-UAT.md (по результатам ручного тестирования)

Если всё пройдено → отметь фазу как Complete в ROADMAP.md.
Если есть проблемы → создай план исправлений.
```

## Промпт 7: Возобновление работы (Resume)

```
Прочитай AGENTS.md и .planning/STATE.md.
Прочитай .continue-here.md (если существует).

Покажи:
1. Текущую позицию (фаза, план, задача).
2. Что было сделано в прошлой сессии.
3. Что осталось.
4. Рекомендуемое следующее действие.

Удали .continue-here.md после восстановления контекста.
```

## Промпт 8: Пауза работы (Pause)

```
Создай .continue-here.md по шаблону
phase-forge-playbook/templates/continue-here.md.

Включи:
- Точную позицию (фаза, задача, статус).
- Что сделано в этой сессии.
- Что осталось.
- Принятые решения с обоснованием.
- Блокеры (если есть).
- Контекст (что держал в голове).
- Конкретное следующее действие.

Обнови STATE.md.
```

## Промпт 9: Отладка (Debug)

```
Прочитай AGENTS.md.

Проблема: <описание>

Создай сессию отладки в .planning/debug/[slug].md
по шаблону phase-forge-playbook/templates/DEBUG.md.

1. Собери симптомы (expected, actual, errors, reproduction).
2. Сформулируй 2-3 гипотезы.
3. Проверяй по одной, фиксируя evidence и eliminated.
4. Минимальный фикс + регрессионный тест.

Если 3 гипотезы отвергнуты подряд — остановись и спроси.
```

## Промпт 10: Автономное выполнение фазы (end-to-end)

```
Прочитай AGENTS.md и .planning/STATE.md.
Прочитай .planning/ROADMAP.md — фаза [N].
Прочитай CONTEXT.md для фазы (если есть).

Выполняй фазу [N] полностью:

1. Создай планы (если ещё не созданы):
   - Исследование → PLAN.md файлы.

2. Выполняй планы по волнам:
   - Каждая задача: diff → verify → коммит.
   - SUMMARY.md после каждого плана.

3. Верифицируй:
   - Goal-backward проверка vs Success Criteria.
   - Полный тест-suite.

4. Обнови артефакты:
   - STATE.md, ROADMAP.md, CHANGELOG.md.

5. Подготовь UAT список для ручной проверки.

Остановись если:
- Тесты падают 2 раза подряд по разным причинам.
- Нужно решение по архитектуре/зависимостям.
- Задача выходит за scope фазы.
```

## Промпт 11: Аудит милестоуна

```
Прочитай .planning/REQUIREMENTS.md и .planning/ROADMAP.md.
Проверь все VERIFICATION.md и UAT.md файлы.

Проведи аудит:
1. Все v1 requirements покрыты? Список uncovered.
2. Все фазы прошли верификацию? Список incomplete.
3. Есть ли стабы в кодовой базе? Поиск: TODO, NotImplementedError, pass.
4. Все тесты проходят?

Результат:
- Покрытие: X из Y requirements complete.
- Gaps: список незакрытых требований.
- Рекомендация: ready to ship / needs more work.

Если есть gaps → предложи план закрытия (дополнительные фазы).
```

## Промпт 12: Quick Mode (ad-hoc задача)

```
Прочитай AGENTS.md.

Задача: <описание>

Выполни с гарантиями Phase Forge:
1. Мини-план (3-5 шагов).
2. Реализация с минимальным diff.
3. Тесты (если применимо).
4. Атомарный коммит.
5. Обнови CHANGELOG.md.

Без полной фазовой обвязки, но с quality gates:
- Линтер чистый.
- Тесты зелёные.
- Коммит атомарный.
```
