# Роадмап: ROADMAP.md

ROADMAP.md — план работ по фазам с критериями успеха и прогрессом.
Это "карта путешествия" от идеи до релиза.

---

## Структура фаз

### Нумерация
- **Целые фазы** (1, 2, 3): запланированная работа милестоуна.
- **Десятичные фазы** (2.1, 2.2): срочные вставки (помечаются INSERTED).

Порядок выполнения: 2 → 2.1 → 2.2 → 3 → 3.1 → 4

### Глубина планирования
- **Quick** (3–5 фаз): для прототипов и мелких проектов.
- **Standard** (5–8 фаз): для типичных проектов.
- **Comprehensive** (8–12 фаз): для production-критичных проектов.

## Анатомия фазы

```markdown
### Phase 3: [Название]
**Goal**: Что эта фаза доставляет
**Depends on**: Phase 2
**Requirements**: [REQ-01, REQ-02, REQ-03]
**Success Criteria** (что должно быть TRUE):
  1. Пользователь может [действие] (наблюдаемое поведение)
  2. [Компонент] работает/существует (наблюдаемое поведение)
  3. [Поведение] при [условии] (наблюдаемое поведение)
**Plans**: 3 plans

Plans:
- [ ] 03-01: Краткое описание первого плана
- [ ] 03-02: Краткое описание второго плана
- [ ] 03-03: Краткое описание третьего плана
```

## Правила

### Критерии успеха
- 2–5 наблюдаемых поведений на фазу (с точки зрения пользователя).
- Перекрёстная проверка с requirements при создании роадмапа.
- Перетекают в `must_haves` при планировании фазы.
- Проверяются верификатором после выполнения.

### Планы
- 2–3 плана на фазу (разделяй если > 3 задач или несколько подсистем).
- Нумерация: `{phase}-{plan}-PLAN.md` (например, `01-02-PLAN.md`).
- Количество планов может быть "TBD" на этапе роадмапа.

### Зависимости
- Явно указывай "Depends on" для каждой фазы.
- Фазы без зависимостей теоретически можно выполнять параллельно.

## Прогресс

```markdown
| Phase | Plans Complete | Status | Completed |
|-------|---------------|--------|-----------|
| 1. Foundation | 3/3 | Complete | 2025-01-20 |
| 2. Features | 1/2 | In progress | - |
| 3. Polish | 0/2 | Not started | - |
```

Статусы: `Not started`, `In progress`, `Complete`, `Deferred`.

## Управление scope

### Добавление фазы
Append к концу роадмапа. Используй для расширения scope.

### Вставка фазы (срочное)
Десятичная нумерация: Phase 2.1 вставляется между Phase 2 и Phase 3.
Помечается "(INSERTED)".

### Удаление фазы
Удаляй только будущие фазы (не начатые). Перенумеруй оставшиеся.

## После завершения милестоуна

Реорганизуй роадмап с группировкой по милестоунам:
- Завершённые милестоуны → свернуть в `<details>`.
- Текущий милестоун → развёрнут.
- Непрерывная нумерация фаз (никогда не перезапускай с 01).
