# Phase Forge — Фазовая разработка с ИИ от идеи до релиза

Практический плейбук для разработки проектов с помощью ИИ-агентов.
Основан на методологии **Get Shit Done (GSD)** — системе контекстной инженерии
и спек-ориентированной разработки.

**Ключевая проблема, которую решает:** деградация контекста (context rot) —
падение качества по мере заполнения контекстного окна ИИ.

**Решение:** фазовый цикл с изолированными контекстами, верификацией на каждом этапе
и структурированными артефактами.

---

## Быстрый старт

### 1. Инициализация проекта

Подготовь описание идеи (PRD, заметки, мысли), затем запроси у агента
инициализацию проекта по шаблонам из этого плейбука:

```
Инициализируй проект по phase-forge-playbook.
Вот описание проекта: <описание>
Создай PROJECT.md, REQUIREMENTS.md, ROADMAP.md и STATE.md
в директории .planning/ по шаблонам из phase-forge-playbook/templates/.
```

### 2. Цикл разработки (для каждой фазы)

```
discuss → plan → execute → verify
```

1. **Discuss** — обсуди реализацию фазы, зафиксируй решения в CONTEXT.md.
2. **Plan** — создай детальные планы (PLAN.md) с задачами и верификацией.
3. **Execute** — выполни планы волнами (параллельно где возможно).
4. **Verify** — проверь результат: автотесты + ручной UAT.

### 3. Завершение

```
audit → complete-milestone → new-milestone (при необходимости)
```

## Как пользоваться этим плейбуком

### Подготовка (один раз на проект)

1. **Создай описание проекта** — PRD, идея, заметки.
2. **Попроси агента инициализировать проект:**
   - Используй промпт из [`12-prompt-recipes.md`](./12-prompt-recipes.md) (Промпт 1: Инициализация).
   - Агент создаст `PROJECT.md`, `REQUIREMENTS.md`, `ROADMAP.md`, `STATE.md`.
   - Одобри роадмап.
3. **Создай `AGENTS.md`:**
   - Попроси агента создать `AGENTS.md` на основе шаблона из [`templates/AGENTS.md`](./templates/AGENTS.md).
   - Подстрой под специфику проекта.

### Работа над фазой (повторяй для каждой фазы)

1. **Discuss фаза** — зафиксируй свои предпочтения:
   - Используй промпт "Обсуждение фазы" из [`12-prompt-recipes.md`](./12-prompt-recipes.md).
   - Результат: `CONTEXT.md` для фазы.

2. **Plan фаза** — создай план выполнения:
   - Используй промпт "Планирование фазы" из [`12-prompt-recipes.md`](./12-prompt-recipes.md).
   - Результат: один или несколько `PLAN.md` с задачами.

3. **Execute фаза** — выполняй задачи:
   - Используй промпт "Выполнение фазы" из [`12-prompt-recipes.md`](./12-prompt-recipes.md).
   - Агент выполняет планы, коммитит по задачам, обновляет STATE.md.

4. **Verify фаза** — проверь результат:
   - Используй промпт "Верификация фазы" из [`12-prompt-recipes.md`](./12-prompt-recipes.md).
   - Пройди UAT, зафиксируй баги.
   - При проблемах: diagnose → fix → re-verify.

5. **Переход к следующей фазе:**
   - Обнови STATE.md и ROADMAP.md.
   - Очисти контекст (`/clear` в Claude Code).
   - Повтори цикл.

### Между сессиями

- **Пауза:** используй промпт "Пауза работы" → создаётся `continue-here.md`.
- **Возобновление:** используй промпт "Возобновление работы" → контекст восстанавливается.

### Завершение милестоуна

1. **Аудит:** проверь все требования → все фазы выполнены?
2. **Complete:** архивируй милестоун, создай тег.
3. **Новый милестоун:** при необходимости — новый цикл требований и фаз.

## Структура артефактов проекта

```
.planning/
  PROJECT.md              # Видение, ценность, ограничения, решения
  REQUIREMENTS.md         # Требования v1/v2 с трассировкой по фазам
  ROADMAP.md              # Фазы, критерии успеха, прогресс
  STATE.md                # Текущая позиция, решения, метрики
  MILESTONES.md           # Архив завершённых милестоунов
  config.json             # Настройки воркфлоу
  phases/
    XX-phase-name/
      XX-CONTEXT.md       # Решения по реализации (от discuss)
      XX-RESEARCH.md      # Исследование экосистемы (от plan)
      XX-YY-PLAN.md       # Атомарные планы выполнения
      XX-YY-SUMMARY.md    # Результаты выполнения
      XX-VERIFICATION.md  # Верификация фазы
      XX-UAT.md           # Результаты ручного тестирования
  debug/                  # Активные сессии отладки
    resolved/             # Архив решённых проблем
  todos/
    pending/              # Захваченные идеи
    done/                 # Выполненные задачи
AGENTS.md                 # Контракт агента (в корне репозитория)
CHANGELOG.md              # Лог изменений
```

## Отличия от классического ai-playbook

| Аспект | ai-playbook | Phase Forge |
|--------|-------------|-------------|
| Единица работы | PR (≤ 3 файла, ≤ 200 строк) | Фаза (2-3 плана с задачами) |
| Контекст | Спека + policy в промпте | Структурированные артефакты (PROJECT, REQUIREMENTS, STATE) |
| Деградация контекста | Не решается явно | Свежий контекст для каждого плана |
| Параллелизм | Нет | Волновое выполнение (независимые планы параллельно) |
| Верификация | Чек-лист + тесты | 3 уровня: автотесты → goal-backward → UAT |
| Отладка | Минимальный фикс | Персистентная сессия (DEBUG.md) |
| Между сессиями | handoff.md | STATE.md + continue-here.md |
| Масштаб | Задача → PR | Идея → Фазы → Милестоун → Релиз |

## Быстрый чек-лист старта

1. Опиши идею проекта.
2. Попроси агента создать PROJECT.md, REQUIREMENTS.md, ROADMAP.md.
3. Одобри роадмап.
4. Для первой фазы: discuss → plan → execute → verify.
5. Между фазами: обнови STATE.md, очисти контекст.
6. В конце сессии: создай continue-here.md.
