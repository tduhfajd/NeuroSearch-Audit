# Glossary

## Core Terms
- Smart Crawler: модуль обхода сайта, строящий структуру, link/entity graph и исходные факты для анализа.
- Semantic Crawl: слой извлечения смысловых блоков страницы (definition, pricing, faq, proof и т.д.).
- Entity Graph: граф сущностей сайта и связей между ними (organization, service, price, geo, document, faq).
- Coverage Gap: отсутствующий обязательный блок/атрибут для выбранного типа страницы или сущности.
- Contradiction: конфликт фактов (например, разные цены/сроки/условия на разных страницах).

## Visibility Dimensions
- SEO: видимость сайта в классическом поиске.
- GEO: видимость сайта в генеративном поиске и AI-ответах.
- AEO: пригодность контента для ответных блоков и формата question-answer.
- AIO: корректность понимания сайта AI-моделями без искажений.
- LEO: пригодность сайта для agent/recommendation сценариев (подобрать, сравнить, выбрать).

## Scoring
- ECS (Entity Coverage Score): полнота обязательных блоков по сущности.
- CS (Contradiction Score): взвешенная оценка конфликтов.
- ARS (AI Readiness Scores): набор оценок по SEO/GEO/AEO/AIO/LEO.
- LVI (Lead Value Index): оценка коммерческого потенциала объема работ.

## Output Artifacts
- site_data.json: агрегированные данные страниц и технические атрибуты.
- site_graph.json / entity_graph.json: структура связей страниц и сущностей.
- page_blocks.json: нормализованные блоки контента.
- coverage_report.json: непокрытые обязательные элементы.
- contradictions.json: найденные конфликты фактов.
- ai_readiness_scores.json: оценки готовности по измерениям видимости.
- recommendations.json: нормализованные рекомендации.
- prompts/*: промты для генерации документов внешним AI.

## Document Types
- Клиентский отчет: бизнес-ориентированный аудит и приоритеты.
- Технический отчет: доказательная матрица проблем и критерии приемки.
- План оптимизации: этапы, сроки, зависимости.
- Технический план работ/backlog: задачи формата Jira/GitLab.
- Коммерческое предложение: пакеты работ и сметная логика.

## Facts
