# Document Templates

## D-001 Клиентский отчет (продающий)
- Аудитория: владелец бизнеса/директор.
- Объем: 5-12 страниц (MVP), расширяемо.
- Структура:
1. Executive summary.
2. Где теряется видимость и заявки.
3. Топ-10 проблем по приоритету.
4. Быстрые победы (quick wins).
5. Дорожная карта улучшений.
6. Ожидаемый эффект (как гипотеза).
- Обязательные элементы: evidence links, список data gaps.

## D-002 Технический отчет
- Аудитория: SEO/техспециалисты.
- Структура:
1. Методика аудита и ограничения.
2. Матрица проблем (URL -> проблема -> влияние -> рекомендация).
3. Contradictions и риски.
4. Критерии приемки по исправлениям.
5. Приложение с артефактами.
- Обязательные элементы: severity, priority, acceptance criteria.

## D-003 План оптимизации
- Аудитория: PM/аккаунт/техлид.
- Структура:
1. Этапы (MVP-подобная последовательность внедрения).
2. Зависимости и риски.
3. Сроки и контрольные точки.
4. Метрики контроля прогресса.
- Обязательные элементы: владелец этапа, вход/выход этапа.

## D-004 Технический план работ (Backlog)
- Аудитория: production team.
- Формат задачи:
1. Title.
2. Context/Problem.
3. Scope.
4. Acceptance Criteria.
5. Evidence links.
6. Priority/Effort.
- Обязательные элементы: трассируемость к finding_id.

## D-005 Коммерческое предложение
- Аудитория: клиент/закупка.
- Структура:
1. Цели проекта.
2. Пакеты услуг (базовый/стандарт/макс).
3. Что входит в каждый пакет.
4. План этапов и ожидаемый результат.
5. Стоимостная логика (привязка к LVI/сложности).
- Обязательные элементы: прозрачные ограничения и допущения.

## Template Quality Rules
- Без неподтвержденных утверждений.
- Явное разделение факта и гипотезы эффекта.
- Использование только approved JSON источников.

## Facts
- `idea.md` фиксирует 5 типов документов как центральный output платформы.

## Assumptions
- У агентства есть базовые бренд-шаблоны, в которые можно маппить структуру.

## Hypotheses
- Единые шаблоны документов ускорят масштабирование пресейла.

## Open Questions
- Нужны ли отдельные шаблоны под разные сегменты клиентов (SMB/enterprise).

## Traceability
- `idea.md`: блоки о “пакете документов”, ролях аудиторий и коммерческом применении результатов анализа.
