# Phase 2, Plan 3: Crawler Integration: API, Queue, Persistence, and End-to-End

## Objective

Интегрировать краулер с API и очередью, обеспечить пакетное idempotent-сохранение в PostgreSQL, обновление статусов аудита и end-to-end сценарий `POST /audits` -> заполнение `pages`.

## must_haves

1. `POST /audits` запускает краулинг домена, после завершения в `pages` сохранены результаты (целевой сценарий: ≥50 страниц).
2. Результаты краулинга стабильно сохраняются в PostgreSQL с upsert по (`audit_id`, `url`).
3. Статусы robots/sitemap и crawl errors фиксируются в `audits.meta`.
