# Requirements: NeuroSearch Audit

**Defined:** 2026-02-26
**Core Value:** Аудит сайта → готовый PDF-отчёт с КП за один запуск

---

## v1 Requirements

Обязательно для первого рабочего инструмента.

### Crawler — Краулер

- [ ] **CRL-01**: Пользователь вводит URL домена → инструмент обходит до 200 страниц сайта, начиная с sitemap.xml (при наличии) или с главной страницы
- [ ] **CRL-02**: Для каждой страницы собирается: HTTP статус, title, h1, meta description, canonical, robots meta, ссылки (внутренние/внешние), JSON-LD разметка, счётчик слов
- [ ] **CRL-03**: Поддержка JS-рендеринга через Playwright (для страниц, где контент загружается через JavaScript)
- [ ] **CRL-04**: Краулер проверяет наличие и корректность `robots.txt` и `sitemap.xml`
- [ ] **CRL-05**: Скорость (Core Web Vitals / PageSpeed score) собирается для топ-10 страниц по входящим ссылкам

### Analyzer — Технический анализатор

- [ ] **ANA-01**: Детектирует страницы, закрытые от индексации (noindex, disallow в robots.txt)
- [ ] **ANA-02**: Находит дубли title и h1 между страницами
- [ ] **ANA-03**: Проверяет длину meta description (норма: 120–160 символов) и title (30–60 символов)
- [ ] **ANA-04**: Детектирует отсутствие или некорректный canonical
- [ ] **ANA-05**: Находит битые ссылки (4xx, 5xx) и цепочки редиректов (3+)
- [ ] **ANA-06**: Проверяет наличие Schema.org разметки: Organization, FAQPage, HowTo, Article, BreadcrumbList
- [ ] **ANA-07**: Валидирует структуру JSON-LD (обязательные поля для каждого типа schema)
- [ ] **ANA-08**: Проверяет наличие коммерческих сигналов доверия (страницы: контакты, о компании, политика, реквизиты)

### AI Bridge — ChatGPT анализ

- [ ] **AIB-01**: Playwright сессия с chat.openai.com сохраняется после первой ручной авторизации (storage_state.json)
- [ ] **AIB-02**: Инструмент автоматически отправляет текст топ-10 страниц в ChatGPT и получает структурированную оценку
- [ ] **AIB-03**: ChatGPT оценивает каждую страницу по 5 параметрам: Answer Format, Structure Density, Definition Coverage, Authority Signals, Schema Need — возвращает JSON с оценками и рекомендациями
- [ ] **AIB-04**: При истёкшей сессии инструмент уведомляет пользователя и запрашивает повторную авторизацию
- [ ] **AIB-05**: Очередь запросов с задержкой (≥15 сек между запросами) для соблюдения rate limit ChatGPT Plus

### Scoring — Скоринг

- [ ] **SCR-01**: Рассчитывается SEO Tech Health Score (0–100) на основе взвешенных rule-based проверок
- [ ] **SCR-02**: Рассчитывается AI Visibility Readiness Index (AVRI, 0–100) по формуле: `(Answer×0.25) + (Structure×0.20) + (Definition×0.20) + (Authority×0.20) + (Schema×0.15)`
- [ ] **SCR-03**: Каждая проблема классифицируется по приоритету: P0 (критично) / P1 (важно) / P2 (AI-готовность) / P3 (улучшение)

### Dashboard — Интерфейс

- [ ] **UI-01**: Стартовая страница со списком всех аудитов (домен, дата, SEO score, AVRI, статус)
- [ ] **UI-02**: Форма запуска нового аудита: URL, ниша/регион, цель (лиды / инфо / локал / B2B), глубина краулинга (50/200/500)
- [ ] **UI-03**: Страница результатов аудита: три секции — Tech Health, AI Readiness, Issue Map
- [ ] **UI-04**: Прогресс-бар краулинга в реальном времени (websocket или polling)

### Reports — Отчёты и КП

- [ ] **REP-01**: Генерация PDF-отчёта с разделами: Executive Summary, Tech Health, AI Readiness, Issue Map, Recommendations
- [ ] **REP-02**: Числа и факты в отчёте берутся из детерминированного анализатора (не из LLM)
- [ ] **REP-03**: Текстовые описания проблем и рекомендации генерируются ChatGPT на основе данных аудита
- [ ] **REP-04**: Генерация КП с 3 пакетами (Start / Growth / Authority) — автоматический выбор релевантного пакета на основе карты проблем

## v2 Requirements

Следующий релиз после MVP.

### Competitor Analysis

- **CMP-01**: Анализ 3–5 конкурентов по тем же метрикам, сравнительная таблица
- **CMP-02**: Автоматический подбор конкурентов по ключевым словам ниши

### Advanced Scoring

- **ADV-01**: Полный AVRI для всех страниц сайта (не только топ-10)
- **ADV-02**: Выявление "сиротских страниц" (нет входящих внутренних ссылок)
- **ADV-03**: Тематические кластеры — карта покрытия интентов

### KP Automation

- **KP-01**: Расчёт стоимости пакетов по формуле (число страниц × ставка + объём контентных работ)
- **KP-02**: Экспорт КП в DOCX

### Queue & Monitoring

- **QUE-01**: Очередь аудитов — несколько сайтов подряд без ручного запуска
- **MON-01**: Еженедельный автоматический повторный краулинг с diff изменений

## Out of Scope

| Feature | Причина |
|---------|---------|
| SaaS / облачный деплой | Внутренний инструмент, не продукт для продажи |
| API-мониторинг позиций SERP | Нестабильно, требует дорогих API, не в core value |
| Гарантия "попадания в ChatGPT" | Технически невозможно гарантировать |
| Интеграция Google Search Console / Яндекс.Вебмастер API | Усложняет onboarding, не критично для MVP |
| Мобильная версия интерфейса | Один пользователь, только Mac |
| Командная работа / роли / доступы | Одиночный инструмент |
| Анализ ссылочного профиля (backlinks) | Требует внешних API (Ahrefs/Serpstat), v3 задел |
